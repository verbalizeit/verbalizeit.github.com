Developer API Resources: Non-Live Workflows


Introduction

Workflows are the blueprint used to define the language and data transformations that occur to source asset files (text or audio files) as they make their way though the VerbalizeIt API. They define source and target languages, file input/output formats, and custom instructions for our translator community. Workflows can be created and managed through the developer portal and each application can have its own custom workflows.

Worflow Parameters:

  • Source Language
  • Target Language
  • Source Gender
  • Target Gender
  • Source File (audio, text, structured text)
  • Target File (audio, text, structured text)
  • Custom Instructions (optional)
